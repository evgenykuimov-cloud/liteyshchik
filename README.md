# ООО «Камский Литейный Завод» — корпоративный сайт-каталог

Многостраничный B2B-сайт на Next.js для литейной продукции, мехобработки и
запросов на изготовление деталей по чертежам заказчика.

## Стек

- Next.js App Router, React, TypeScript strict
- Tailwind CSS
- Zod для серверной валидации
- Metadata API, sitemap, robots, JSON-LD
- Локальная типизированная модель каталога

## Запуск

```bash
npm install
npm run dev
npm run lint
npm run typecheck
npm run build
npm run start
```

## Структура

- `src/app` — страницы, Route Handlers и SEO-маршруты
- `src/components` — интерфейсные компоненты
- `src/data` — товары, категории, документы и контент
- `src/types` — типы предметной области
- `src/lib` — схемы, адаптеры, rate limit и аналитика
- `src/config/company.ts` — данные компании и реквизиты
- `public/images` — изображения
- `public/documents` — подтверждённые документы

## Настройка

Контакты и реквизиты меняются в `src/config/company.ts`. Товары находятся в
`src/data/catalog.ts`, документы — в `src/data/documents.ts`.

Фотографии изделий и часть документов пока помечены как TODO. Их нужно заменить
только подтверждёнными материалами предприятия: реальными фотографиями,
сертификатами, лицензиями, благодарственными письмами, PDF-техописаниями и
чертежами для проектировщиков.

Для реальной отправки заявок нужно реализовать `LeadRepository`,
`NotificationService`, `CRMAdapter` и `FileStorage` из `src/lib/adapters.ts`,
затем установить `LEAD_DELIVERY_MODE=configured`. Сейчас форма валидирует запрос,
но честно возвращает статус отсутствия настроенного канала.

Переменные окружения описаны в `.env.example`. Аналитика подключается через
адаптер `src/lib/analytics.ts`; идентификаторы не включены в код.

Логотип реализован компонентом `src/components/logo.tsx`. Перед публикацией
нужна юридическая проверка политики конфиденциальности и согласия на обработку
персональных данных.
