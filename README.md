# ООО «Литейщик» — корпоративный сайт-каталог

Многостраничный B2B-сайт на Next.js для каталога чугунных люков,
дождеприёмников, обойм, крышек, решёток и нестандартного литья.

## Стек

- Next.js App Router, React, TypeScript strict
- Tailwind CSS
- Zod для серверной валидации
- Metadata API, sitemap, robots, JSON-LD
- Локальная типизированная модель каталога

## Запуск

```bash
npm install
npm run dev
npm run lint
npm run typecheck
npm run build
npm run start
```

## Структура

- `src/app` — страницы, Route Handlers и SEO-маршруты
- `src/components` — интерфейсные компоненты
- `src/data` — демонстрационные товары, категории, документы и контент
- `src/types` — типы предметной области
- `src/lib` — схемы, адаптеры, rate limit и аналитика
- `src/config/company.ts` — данные компании
- `public/images` — изображения
- `public/documents` — подтверждённые документы

## Настройка

Контакты меняются в `src/config/company.ts`. Товары находятся в
`src/data/catalog.ts`, документы — в `src/data/documents.ts`.

Все текущие товарные записи и изображение товара являются демонстрационными.
Размеры, масса, марки чугуна, классы нагрузки, стандарты, цены, наличие и сроки
не вымышлены: до получения данных показывается «Уточняется».

Для реальной отправки заявок нужно реализовать `LeadRepository`,
`NotificationService`, `CRMAdapter` и `FileStorage` из `src/lib/adapters.ts`,
затем установить `LEAD_DELIVERY_MODE=configured`. Сейчас форма валидирует запрос,
но честно возвращает статус отсутствия настроенного канала.

Переменные окружения описаны в `.env.example`. Аналитика подключается через
адаптер `src/lib/analytics.ts`; идентификаторы не включены в код.

Логотип реализован компонентом `src/components/logo.tsx` и заменяется без
перестройки layout. Перед публикацией запросите материалы из файлов требований
в корне проекта и проведите юридическую проверку текстов политики.
